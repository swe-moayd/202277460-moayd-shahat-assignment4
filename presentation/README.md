# Assignment 4 Presentation Plan

This outline is prepared for the required 5-7 minute professional demonstration.

Supporting files:

- `presentation/demo-script.md`: full speaking script with timing.
- `presentation/slides.html`: browser-based slide deck that can be presented or printed to PDF.

## 1. Introduction - 1 Minute

- Introduce the project as the final Assignment 4 portfolio.
- Explain that it is an incremental upgrade from Assignment 3.
- State the main objective: combine portfolio content, JavaScript features, API integration, documentation, and professional presentation.

## 2. Technical Demonstration - 3 to 4 Minutes

- Show the hero section, navigation, responsive layout, and visual design.
- Demonstrate theme persistence by toggling theme and reloading.
- Save a visitor name and show the personalized greeting.
- Toggle availability status.
- Use project search, category filter, complexity filter, and sorting.
- Point out live project statistics.
- Show the GitHub API section and explain the fallback state.
- Submit invalid and valid contact form examples.

## 3. Technical Deep Dive - 1 to 2 Minutes

- Explain the `state` object and `localStorage` usage.
- Explain `getFilteredProjects()` as the shared filtering and sorting source.
- Explain API handling with `fetch()` and `AbortController`.
- Explain validation logic and accessible feedback.
- Discuss the incremental upgrade from Assignment 3 to Assignment 4.

## 4. AI Integration Highlights

- Mention ChatGPT / Codex use for planning, implementation support, debugging, and documentation.
- Mention AI image generation for the hero asset.
- Explain that AI output was reviewed, modified, and tested before inclusion.
- Reference `docs/ai-usage-report.md`.

## 5. Challenges and Lessons Learned

- Challenge: keeping the project polished without adding unnecessary frameworks.
- Challenge: making the GitHub API section graceful when network requests fail.
- Challenge: updating documentation so it accurately matches the final app.
- Lesson: incremental development is easier to explain and maintain than rebuilding from scratch.

## 6. Conclusion - 1 Minute

- Summarize the final outcomes: polished portfolio, working interactive features, API integration, validation, documentation, and presentation plan.
- Mention possible future improvements such as backend contact submission, more project case studies, and final deployment link.

## Final Video Checklist

- Keep the recording between 5 and 7 minutes.
- Use a large enough browser window and readable zoom level.
- Test the site immediately before recording.
- Show both the application and the documentation files.
- Submit the final video file through the required course submission channel.
