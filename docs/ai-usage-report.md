# AI Usage Report

## Tools Used and Use Cases

### ChatGPT / Codex

- Used to analyze the existing Assignment 2 portfolio structure
- Used to generate and refine JavaScript logic for filtering, sorting, state management, and validation
- Used to improve the HTML structure and CSS organization
- Used to draft clearer documentation for the README and technical report

## Benefits

- Helped accelerate implementation by turning the Assignment 3 requirements into a structured feature set
- Made it easier to identify missing rubric items such as API failure handling, stronger documentation, and user-state persistence
- Supported debugging and cleanup by suggesting more consistent file organization and logic flow

## Challenges and Limitations

- AI-generated suggestions still required review because some ideas were too generic for the assignment rubric
- Not every suggestion matched the exact project context, so changes had to be adapted manually
- UI and wording recommendations needed revision to better match the assignment requirements and portfolio tone

## Learning Outcomes

- Learned how to structure a small front-end project around explicit application state
- Improved understanding of integrating public APIs with defensive error handling
- Practiced turning assignment requirements into implementable features with clear separation between HTML, CSS, and JavaScript
- Improved documentation quality by explaining design choices, logic, and validation rules clearly

## Responsible Use and Modifications

- All AI suggestions were reviewed before being accepted
- Generated code and text were modified to fit the actual project structure and assignment goals
- Validation logic, project data, UI text, and documentation were adjusted manually for correctness and originality
- The final submission reflects human review, editing, and testing rather than unverified AI output
